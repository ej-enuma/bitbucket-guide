🌿 브랜치 & 머지

브랜치는 본체(main)를 건드리지 않고 따로 작업하는 갈래예요. 작업이 끝나면 다시 합칩니다(머지).

1. 새 브랜치 만들고 이동

$ git switch -c feature/login

-c는 새로 만들면서 이동한다는 뜻이에요. 이제 이 갈래에서 자유롭게 작업하세요.

2. 작업 후 올리기

$ git add .
$ git commit -m "로그인 기능 추가"
$ git push origin feature/login

3. main으로 합치기 (merge)

먼저 main으로 이동해 최신 상태로 만든 뒤, 브랜치를 합칩니다.

$ git switch main
$ git pull origin main
$ git merge feature/login
💡 실무에서는 직접 머지보다 Pull Request로 리뷰를 받고 합치는 경우가 많아요. 다음 페이지에서 다룹니다.

4. 충돌(conflict)이 났다면?

같은 줄을 두 사람이 다르게 고치면 충돌이 나요. 당황하지 말고, 충돌난 파일을 열면 이렇게 표시돼요.

<<<<<<< HEAD
내가 쓴 코드
=======
팀원이 쓴 코드
>>>>>>> feature/login

표시 기호(<<<, ===, >>>)를 지우고 올바른 최종 코드만 남긴 뒤, 다시 add → commit 하면 해결됩니다.
